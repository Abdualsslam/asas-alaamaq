# المطلوب في هذه المرحلة

أريد منك تأسيس مشروع Landing Page حديث واحترافي فقط، بدون الدخول العميق في كل المحتوى النهائي الآن.

الهدف من هذه المرحلة:
بناء أساس تقني وتصميمي نظيف نستطيع بعده إضافة الأقسام والأنيميشن والمحتوى التفصيلي.

---

# التقنية المطلوبة

استخدم:

- Next.js App Router
- TypeScript
- Tailwind CSS
- Framer Motion
- GSAP
- Lenis smooth scroll
- lucide-react للأيقونات
- clsx و tailwind-merge إن احتجت
- بدون Backend
- بدون CMS
- بدون API حقيقي

المشروع صفحة هبوط واحدة فقط في البداية.

---

# اللغة والاتجاه

- اللغة الأساسية: العربية
- الاتجاه: RTL
- يجب أن يكون التصميم عربيًا احترافيًا بالكامل
- اترك قابلية مستقبلية لإضافة اللغة الإنجليزية، لكن لا تبني نظام ترجمة كامل الآن
- لا تستخدم نصوص إنجليزية في الواجهة إلا عند أسماء الخدمات بجانب العربية عند الحاجة

# بنية الصفحة المطلوبة مبدئيًا

أنشئ صفحة Landing Page تحتوي على الأقسام التالية كهيكل أولي:

## 1. Header / Navbar

- Logo
- روابط الأقسام
- زر تواصل معنا

## 2. Hero Section

- عنوان قوي
- نص داعم
- أزرار CTA
- مساحة مخصصة لصورة أو مشهد بصري هندسي
- خلفية فيها خطوط أو طبقات أرض مجردة

## 3. Problem / Risk Section

- يشرح أن الحفر ليس مجرد حفر
- بطاقات مخاطر:
  - تربة غير مستقرة
  - مياه جوفية
  - ضغط جانبي
  - حفريات عميقة
  - منشآت مجاورة

## 4. About Section

- من نحن
- تموضع الشركة كشركة هندسة أرضية وليست مقاول عام

## 5. Services Section

- سند الحفريات
- الشوتكريت
- الميكروبايل
- نزح المياه
- الأنابيب المثقبة

## 6. Execution Approach Section

- نفهم الموقع
- نختار النظام
- ننفذ على مراحل
- نراقب ونسيطر

## 7. Capabilities Section

- العمل في ظروف تربة معقدة
- مواقع حضرية ضيقة
- دمج أنظمة دعم متعددة
- سرعة الاستجابة
- قرارات ميدانية

## 8. Safety & Quality Section

- السلامة كنظام هندسي
- استقرار الحفريات
- سلامة العاملين
- حماية المنشآت المجاورة

- Grid مبدئي للصور
- Placeholders لحين إضافة الصور الحقيقية
- فلترة مستقبلية حسب نوع الخدمة، لكن لا تنفذ الفلترة الآن إلا إذا كانت بسيطة جدًا

## 10. Final CTA Section

- دعنا نناقش متطلبات مشروعك
- زر تواصل عبر واتساب
- زر إرسال بريد

## 11. Footer

- Logo
- بيانات التواصل
- روابط الأقسام

# متطلبات الواجهة

# جودة الكود

- استخدم TypeScript بشكل صحيح
- لا تكرر النصوص داخل المكونات، اجعل البيانات في ملفات data عند الحاجة
- استخدم Components قابلة لإعادة الاستخدام
- استخدم className منظم وواضح
- لا تضع أي كود غير مستخدم
- لا تستخدم مكتبات كثيرة بدون حاجة
- لا تضع صور خارجية من الإنترنت
- لا تضع TODO عشوائي، فقط TODO واضح للأصول التي سأضيفها لاحقًا
